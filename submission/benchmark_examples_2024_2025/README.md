# Benchmark Examples 2024-2025

这个文件夹是给 `AffordanceBench Studio` 做视觉与软件呈现对标用的轻量材料包。

## 先说结论

你刚发的那张图不算“丑”，但确实**不够像强 OSS 投稿里的主视觉**。主要问题不是清晰度，而是：

1. 更像“证据拼贴”而不是“产品级主图”。
2. 四块内容的信息密度和视觉层级不一致。
3. 左上、右上是终端截图，左下是 JSON 文本，右下是卡片拼图，风格混杂。
4. reviewer 第一眼会感到“材料很多”，但未必会觉得“软件很成熟、界面很强、交互很完整”。

所以这张图可以当 `evidence panel`，但不太像最强对标作品那种“看一眼就知道这是个成体系软件”的主视觉。

## 你接下来最该比较的维度

1. 别人的第一页主视觉是不是更像“软件界面 / 系统架构 / 交互产品”。
2. 别人的 repo 首页是不是更早给出使用故事，而不是先给证据碎片。
3. 别人的软件身份词是不是更稳定，比如 `toolkit / library / platform / engine / framework`。
4. 我们当前这张图是否应该降级为“证据图”，另补一张更强的“主视觉图”。
